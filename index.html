data-uris

说明

data URIs是由RFC 2397定义, 允许将一个小文件进行编码后嵌入到另外一个文档里.

语法


mediatype是一种MIME类型字符串, 例如 “image/jpeg” 代表 JPEG 图像文件.如果省略这个参数, 则默认值为text/plain;charset=US-ASCII

如果数据是文本类型, 你可以直接将文本嵌入在data:后面 (根据文本类型以及编码,使用合适的HTML实体编码或者百分号编码). 如果是二进制数据, 你可以将数据进行base64编码之后再进行嵌入.

### 示例
data:,Hello%2C%20World!
简单的text/plain类型数据
data:text/plain;base64,SGVsbG8sIFdvcmxkIQ%3D%3D
base64编码过的数据
data:text/html,%3Ch1%3EHello%2C%20World!%3C%2Fh1%3E
一个HTML文档源代码

常见问题

1.长度限制
虽然Mozilla支持无限长度的data URIs,但是标准中并没有规定浏览器必须支持无限长度的data URIs.比如,Opera 11限制data URIs的长度最多为65000个字符.

2.缺乏错误处理
MIME类型错误或者base64编码错误,都会造成data URIs无法被正常解析, 但不会有任何相关错误提示.

3.不支持查询字符串
一个data URI的数据字段是没有结束标记的,所以尝试在一个data URI后面添加查询字符串会导致,查询字符串也一并被当作数据字段.例如:

Web Notifications API

用途

web notification 可以定时向客户端推送消息通知，比如Web OA中可以在用户的浏览器中的右下角弹出一个信息提示窗口，即时传达给用户相关通知等。

语法

Methods(方法), Properties(属性), and Events(事件)

methods

  • permission 获得授权状态
    Notification.permission会获得以下三个字符串之一：default, denied, granted。
    default 相当于“拒绝”，这表示用户没有选择网站给出的通知。
    denied 表示拒绝，这意味着用户不希望通知。
    granted 表示用户同意进行桌面通知。
    当 Notification.permission 获得 default 或者 denied 后，我们还可以使用 window.Notification.requestPermission(); 在用户下次进入此页面时再一次询问是否可以进行桌面通知。
  • requestPermission() 请求用户授权
  • close() 关闭通知

Properties

Notification(title, options)

  • title 消息标题
  • options JSON数据，有以下选项
    body 消息的主体内容，文本类型
    lang 消息文本的语言，其值必须符合的BCP47标准，如 en-US, zh-CN
    dir 消息文本的方向，有rtl & ltr
    tag 设置消息的Tag归类，以便检索、替换、删除通知
    icon 消息窗口的ICON URL

Events

  • onshow 对话框展示时触发
  • onclick 当用户点击通知对话框时，触发
  • onclose 当用户关闭对话框，或脚本调用close方法时，触发
  • onerror 错误事件，目前来看一般只会接触到权限错误

Media Fragments

用途

Media Fragments让视频、音频等媒体有了可以展示区块（fragments）的功能。这里的裁剪可以是时间的裁剪，也可以是视窗大小的裁剪。

语法

如下，只需要在视频源末尾加上”#”或者”?”，之后加上相应参数，就可以实现裁剪展示功能。

其他属性：
1. t=a,b; 表示a到b之间的时间。若只有a，则表示从a开始播到最后，若只有b，则表示从视频开头到时间b。

2. xywh=160,120,320,240; 表示起始坐标(x,y)，宽度高度(w,h），会截取屏幕的某个区域作为视窗展示视频。也可以定义单位，例如：
xywh=pixel:160,120,320,240;（像素）
xywh=percent:25,25,50,50;（百分比）

3. track=’ogg_1’，表示视频轨道、track=’ogg_2’，表示音频轨道。

4. id=’aaaa’，表示如果文件定义了fragments（区块）的话可以使用他们的对应id值。

dataset

用途

HTMLElement.dataset 属性允许我们访问所有在元素上自定义的data属性 (这种属性一般以data-开头）. 它的结构是一个DOMString映射表，对每一个自定义的数据属性都有一个实体与之对应。

自定义的数据属性名称是以 data- 开头的。 它必须要遵循 the production rule of xml names 规则，该规则是说它只可以包含字母，数字和下面的字符： dash (-), dot (.), colon (:), underscore (_)。此外不应包含ASCII 码大写字母。
